**验证**：`python -m scripts.health` → 🟢 HEALTHY, backend=sentence-transformers

### 2. 注册 Windows 登录 Run 键（开机自动入库）

**注册表项**：`HKCU\...\Run\WorkBuddy-RAG-Bootstrap`

**效果**：开机登录后自动跑入库 + 蒸馏兜底 + D 方案注入，把 `~/.workbuddy/memory/*.md` 同步到 RAG 索引并回写 MEMORY.md。

### 3. 同进程 RAG 集成 Demo

**文件**：`scripts/demo_integration.py`（+148 行）

**关键决策**：单机单用户场景下，**同进程 import 优于 HTTP server**（省 RAM + 启动简单 + 零运维）。

### 4. rag_search skill 升级：subprocess 隔离模式

**架构**：`main.py (任意 python) → subprocess(60s 超时) → venv python → worker.py`

三环境实测全通过：系统 python / managed venv / 裸 `python`。

### 5. Skill 镜像到项目 + 路径自动发现

user-level skill 不在 GitHub 仓库里，换机器就丢。改为自动发现项目根，零个人路径残留，开源无风险。

### 6. bge-m3 + bge-reranker 双模型下载

| 模型 | 大小 | 用途 |
|---|---|---|
| BAAI/bge-m3 | 4.3 GB | 嵌入（1024 维中英双语） |
| BAAI/bge-reranker-v2-m3 | 2.2 GB | Cross-Encoder 重排 |

### 7. 蒸馏兜底 + 时间衰减 + access_count（v0.2.4）

- **蒸馏兜底**：`ingest_wb_memory_oneshot.py` 检查上次蒸馏时间，过期自动补跑（解决电脑不开机时 03:00 cron 漏跑）
- **时间衰减**：`retriever.py` 默认 `enable_decay=True`，公式 `score = rrf × exp(-Δt/τ) × log(1+acc) × (0.7+0.3×conf)`
- **access_count**：检索 Top-K 自动递增，蒸馏时用 conf 作质量代理

### 8. dedup source 层级守门（v0.2.4.1）

**动机**：D 方案实施前，防止 MEMORY.md 骨架与工作日志血肉互相覆盖。

| 新 chunk | 已有 chunk | 决策 | 理由 |
|---|---|---|---|
| summary (MEMORY.md) | full (workspace-log) | **SKIP** | 防骨架反向覆盖血肉细节 |
| full (workspace-log) | summary (MEMORY.md) | **MERGE** | 允许骨架被血肉升级 |
| 同层级 | 同层级 | 按 confidence 比较 | 原逻辑 |

**测试**：8 个新单元测试，覆盖所有边界 + 鹏哥实战工作流。

### 9. D 方案：蒸馏骨架自动注入 MEMORY.md（v0.2.5）★ 核心交付

**文件**：`scripts/inject_distilled_to_memory.py`（349 行）

**职责**：把 distill() 输出的 top-N 高分记忆 → 标记区间写入 `~/.workbuddy/MEMORY.md`

**设计要点**：
1. 标记区间 `<!-- AUTO_INJECT_START/END -->` 隔离手写与自动注入
2. 字符预算三道关卡：全局 2200 + 单项目 500 + 单 chunk 80
3. 项目按 Σscore 倒序排（热门优先）
4. 重复跑幂等：实测 181 → 2471 → 2458 → 2458 稳定
5. ingest_wb_memory_oneshot 串联：蒸馏兜底后自动 inject

**D 与 rag_search 互补关系**：

| 维度 | D 方案 | rag_search |
|---|---|---|
| 触发时机 | WorkBuddy 启动（oneshot） | 用户提问时按需调 |
| 数据来源 | distill() 顶层聚合 | retriever 全库搜索 |
| 输出形式 | 持久 markdown（标记区间） | 临时 JSON 返回 |
| 角色定位 | 启动预热（L2 长期） | 临时召回（对话上下文） |

**测试**：19 个单元测试 PASSED（truncate / assemble / build_inject_block / read_split / 完整流程幂等 / 手写部分保护）

## 四、关键架构决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| 嵌入模型 | bge-m3 | 中文支持好，1024 维，多语言 |

| skill 入口模式 | subprocess 隔离 | 任意 python 都能调 |
| 默认 rerank | 关闭（懒加载） | 默认省 2.3 GB RAM |
| HF_HUB_OFFLINE | 默认开启 | 防止启动时联网 |
| 蒸馏调度 | 兜底机制 | Task Scheduler 漏跑时自动补 |
